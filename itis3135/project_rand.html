Project Overview:

  • The application is all about swimming. The purpose is to provide an overview of the basics of swimming as well as some interesting information that could be useful to know.
  • The intended users of the website are people who want to know more about the sport.
  • The website will contain a homepage, a page about the 4 strokes and their basics, a page about local swimming pools in Charlotte, an about me page, and a page about helping new swimmers get started swimming.

Wireframe:

Site Map:

Home - Page Design:

  • The purpose of this page is to be the landing page and have links to other pages.
  • The users of this page are customers.
  • The content on this page will be the links to different pages as well as some basic information about swimming.
  • This is not a page that will require users to enter data on the page.
  • The page will contain hyperlinks to other pages.

Swimming Basics - Page Design:

  • The purpose of this page is to contain descriptions of the 4 strokes and have pictures/videos of all strokes.
  • The users of this page are customers.
  • The content on this page will be descriptions of strokes, images, and videos showing off the 4 strokes.
  • This is not a page that will require users to enter data on the page.
  • The page will contain hyperlinks to tutorial videos.
  • Special Note: This page will have a small image/video gallery for each stroke explaining the different movements for each stroke.

Local Pools in Charlotte - Page Design:

  • The purpose of this page is to show different pools the user can go to in Charlotte.
  • The users of this page are customers.
  • The content on this page will be different places you can go to swim in Charlotte. The page will also contain information about each location like closing and opening times.
  • This is not a page that will require users to enter data on the page.
  • The page will contain hyperlinks to the different pool websites.
  • Special Note: When the user hovers over a location, the page should show extra info about the place.

Want to Start Swimming? - Page Design:

  • The purpose of this page is to help newcomers get started with swimming.
  • The users of this page are customers.
  • The content on this page will be information about how to get started and will also contain links to different pages that contain useful information.
  • This is not a page that will require users to enter data on the page.
  • The page will contain hyperlinks to swim classes.

About Us - Page Design:

  • The purpose of this page is to contain basic information about the client.
  • The users of this page are customers.
  • The content on this page will be information about the client.
  • This is not a page that will require users to enter data on the page.
  • The page will contain a hyperlink to the client's portfolio.

Contact Us - Page Design:

  • The purpose of this page is to provide useful information that could be used to contact the client.
  • The users of this page are customers.
  • The content on this page will be information about the client as well as an email function to directly email the client.
  • This page will not require users to enter data on the page. However, they will be required to enter data IF they are emailing the client.
  • The page will not contain any hyperlinks or dropdowns.
  • Special Note: This page will contain a built-in email functionality.

Dynamic Functionality:

I intend to have dynamic functionality on multiple pages.

The Swimming Basics page will have an image/video gallery for each stroke explaining the different movements for each stroke. I want to have a gallery for each stroke because that way I can explain the basics of each stroke using pictures and the user may find it more helpful than a block of text.

The Contact Us page will also have a form that can allow a user to send an email directly to the client. I want to have this functionality because I think a page that is dedicated to contacting the client should have some form of sending an email directly to the client.
